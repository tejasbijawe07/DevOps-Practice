
#  DevOps 90 Days Practice

This repository tracks daily hands-on learning and progress towards becoming a **DevOps Engineer**.

---

## 🎯 Objective

* Build strong fundamentals in DevOps tools & practices
* Gain hands-on experience with real-world scenarios
* Document & Track daily learning

---

## 🛠️ Topics Covered

###  Linux

* File System & Navigation
* User & Permission Management
* Process Management
* Networking Commands
* Shell Scripting

###  Git & Version Control

* Git Basics & Workflow
* Branching & Merging
* Pull Requests
* Conflict Resolution

###  Docker

* Containerization Basics
* Docker Images & Containers
* Volumes & Networking
* Docker Compose

###  Kubernetes

* Pods, Deployments, ReplicaSets
* Services & Networking
* ConfigMaps & Secrets
* Ingress & Helm

###  CI/CD

* CI/CD Concepts
* Jenkins Pipelines
* GitHub Actions
* Build & Deployment Automation

###  Cloud (AWS)

* EC2, S3, IAM
* VPC Basics
* Load Balancer
* Deployment on Cloud

###  Infrastructure as Code

* Terraform Basics
* Resource Provisioning
* State Management

###  Monitoring & Logging

* Prometheus
* Grafana
* Logging Basics

---

## 📈 Goal

By the end of 90 days, I aim to:

* Work confidently with DevOps tools
* Build and deploy real-world applications
* Crack DevOps interviews
